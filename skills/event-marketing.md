# Event Marketing — экспертный скилл для event-tech ниши

Этот скилл содержит фреймворки и контекстные знания для продукт-маркетинга в event-management платформе Meet. Используется агентом `event-marketer`.

## 1. ICP Matrix — сегменты ивентов

Не существует «организатора вообще». Всегда работай через сегмент. Каждый сегмент покупает по-разному.

| Сегмент | Кто организатор | Размер ивента | Бюджет на платформу | Решающие факторы | Сезон (TJ/KZ) |
|---|---|---|---|---|---|
| **Свадьбы** | Семья, тамада, ивент-агентство | 100–500 гостей | $30–$150 разовый | Внешний вид приглашения, контроль гостей, статус | Май–сентябрь, пик июль-август |
| **Юбилеи / частные** | Семья, иногда агентство | 30–200 | $10–$50 | Простота, шаблоны, скорость | Круглый год, пики весна/осень |
| **Корпоративы** | HR, ивент-менеджер компании | 50–1000 | $50–$500 (B2B бюджет) | Брендирование, отчёты для бухгалтерии, аккуратные RSVP | Декабрь (НГ), март (Навруз) |
| **Конференции / митапы** | Комьюнити-менеджер, оргкомитет | 100–2000 | $50–$300 | Регистрация, check-in, нетворкинг, аналитика | Осень (сентябрь-ноябрь), весна |
| **Спортивные** | Клубы, лиги | 50–500 | $20–$100 | Командные списки, расписание, статусы | Весна/осень |
| **Концерты / шоу** | Промоутеры | 200–5000 | Билетные комиссии | Билетинг, продажа, аналитика | Круглый год |

**Правило:** при любой рекомендации называй сегмент явно. «Эта фича для свадебного сегмента в KZ» — ок. «Эта фича для организаторов» — НЕ ок.

## 2. JTBD Framework (Jobs To Be Done)

Организаторы не покупают фичи. Они нанимают платформу на конкретную «работу». Формат:

> Когда [ситуация], я хочу [мотивация/действие], чтобы [ожидаемый результат].

### Каталог JTBD по сегментам Meet

**Свадьбы:**
- Когда я готовлю свадьбу на 300 человек, я хочу быстро разослать красивые приглашения с RSVP, чтобы не звонить каждому и точно знать кто придёт.
- Когда тётя меняет решение за день до свадьбы, я хочу мгновенно скорректировать список рассадки, чтобы не путаться в Excel.
- Когда младший брат не зарегистрирован в продукте, я хочу пригласить его по email/телефону без принуждения регистрироваться, чтобы не потерять гостя из-за барьера.

**Корпоративы:**
- Когда HR готовит новогодний корпоратив, я хочу собрать RSVP с подтверждением +1, аллергий и предпочтений еды, чтобы передать в кейтеринг точные числа.
- Когда финдиректор просит отчёт по затратам на ивент, я хочу получить выгрузку списка гостей и статусов, чтобы закрыть документы.

**Конференции:**
- Когда я делаю первую конференцию на 500 человек, я хочу быстро собрать форму регистрации с разными типами участников (спикер/слушатель/спонсор), чтобы не настраивать через 5 разных инструментов.
- Когда гость приходит на check-in, я хочу за 3 секунды отметить его QR-кодом, чтобы не было очереди на входе.

**Виральность (cross-сегмент):**
- Когда я был гостем на хорошей свадьбе через Meet, я хочу легко создать свой ивент в этой же платформе, чтобы не разбираться в новом инструменте.

**Правило:** при оценке фичи спрашивай «какую JTBD это закрывает?». Если ни одной — фича сомнительна для продакт-маркетинга.

## 3. Value Proposition — каноническая формула

```
Для [ICP],
которые [боль/ситуация],
[продукт/фича] — это [категория решения],
который [ключевая выгода].
В отличие от [альтернатива],
мы [ключевое отличие].
```

**Пример (свадебный сегмент TJ):**
> Для семей готовящих традиционную свадьбу на 200–500 гостей, которые тонут в звонках и WhatsApp-уточнениях за неделю до тоя, Meet — это платформа приглашений с RSVP и контролем команды организаторов, которая собирает все ответы гостей в одном месте. В отличие от Excel + WhatsApp-чатов, мы автоматически напоминаем неответившим и обновляем список рассадки в реальном времени.

**Анти-паттерн (так нельзя):**
> «Meet — это удобная платформа для всех типов мероприятий с множеством функций.» — нет сегмента, нет боли, нет отличия. Отклоняй такие формулировки.

## 4. Конкурентное поле

### Глобальные
- **Eventbrite** — билетинг + регистрация. Сильны в B2C ивентах с продажей билетов. Слабо в приватных мероприятиях, нет фокуса на команду организаторов.
- **Luma (lu.ma)** — модерн, удобно, IT-комьюнити. Слабо в локализации (TJ/KZ), нет тяжёлой кастомизации.
- **Meetup** — комьюнити-ивенты, повторяющиеся. Не про разовые свадьбы/корпоративы.
- **Splash / Bizzabo / Cvent** — энтерпрайз-конференции. Дорого, оверкилл для частных ивентов.
- **Hopin / Whova** — гибридные/онлайн конференции. Не для офлайн-свадеб.

### Локальные TJ/KZ
- **TimePad** (RU/KZ) — билетинг, регистрация. Слабо в управлении командой организаторов, нет фокуса на приглашения частных ивентов.
- **WhatsApp + Excel** — это РЕАЛЬНЫЙ конкурент для большинства частных ивентов. Бесплатно и привычно. Чтобы выиграть — нужно явное преимущество (время / точность / статус).
- **Печатные приглашения** — для свадеб всё ещё стандарт. Конкурируем не «технологичностью», а удобством + статусом красивого digital-приглашения.

### Где Meet выигрывает
- Команда организаторов (несколько ролей: OWNER / ADMIN / ADMIN_EVENT) — у большинства конкурентов её нет
- Приглашение незарегистрированных без барьера (PendingInvitation flow)
- Локализация под TJ/KZ (русский, USD/локальные платёжки)
- Виральный invitation loop встроен в продукт

### Где Meet проигрывает (честно)
- Нет продажи билетов (vs Eventbrite/TimePad) — для коммерческих ивентов
- Нет встроенного онлайн-стриминга (vs Hopin) — для гибридных
- Бренд молодой — нет доверия по умолчанию

## 5. Сезонность ивентов в TJ/KZ

Ключевая особенность event-tech: спрос **не равномерный**. Маркетинг-кампания вне сезона — слив бюджета.

| Месяц | Свадьбы | Корпоративы | Конференции | Особые события |
|---|---|---|---|---|
| Январь | низко | Пост-НГ корпоративы | низко | — |
| Февраль | низко | низко | низко | — |
| Март | низко | **ПИК (Навруз)** | средне | Навруз — массовые ивенты |
| Апрель | растёт | средне | растёт | — |
| Май | растёт | низко | средне | — |
| Июнь | **ПИК** | низко | низко | Конец учебного года |
| Июль | **ПИК** | низко | низко | — |
| Август | высоко | низко | низко | — |
| Сентябрь | высоко | растёт | **ПИК** | Начало бизнес-сезона |
| Октябрь | средне | средне | высоко | — |
| Ноябрь | низко | растёт | средне | — |
| Декабрь | низко | **ПИК (НГ)** | низко | Новогодние корпоративы |

**Правило:** запуск маркетинговой инициативы привязывай к сезонному окну. Фича для свадеб — релиз с marketing push в апреле-мае (под пик). Фича для корпоративов — октябрь-ноябрь (под декабрьский пик).

## 6. TJ vs KZ — критические различия

| Параметр | Таджикистан | Казахстан |
|---|---|---|
| Язык лендинга | RU + TJ (опционально) | RU + KZ (важно) |
| Платёжки | Корти Милли, Алиф Mobi, Visa/Mastercard | Kaspi, Halyk, Visa/Mastercard |
| Средний бюджет ивента | Ниже | Выше (в 2-3 раза) |
| Размер свадьбы | 300–700 гостей (масштабнее) | 200–400 |
| Готовность платить за SaaS | Низкая (бесплатное привычно) | Средняя (есть Kaspi-привычка платить онлайн) |
| Конкуренты локальные | Слабые, рынок пустой | TimePad, локальные стартапы |
| Канал лидов | Insta + сарафан + Telegram-каналы | Insta + Kaspi-маркетплейс + Threads |

**Сигнал:** отклик из KZ выше чем из TJ (см. memory: `project_kz_market_signal`). Не значит «забить на TJ» — значит **сейчас инвестировать в KZ-канал сильнее, держа TJ как домашний/тестовый рынок**.

## 7. GTM-каналы — гипотезы для Meet

Приоритизированы по ожидаемому ROI на ранней стадии:

1. **Виральные invitation loops (highest ROI, 0 budget)** — каждый ивент = N гостей видят бренд. Усилить: красивое sharing-превью в WhatsApp/Telegram, банер «создай свой ивент» в email-приглашениях гостю.
2. **Партнёрства с ивент-агентствами / тамада / свадебными салонами** — они организуют десятки ивентов в сезон. Один партнёр = 20-50 ивентов. Модель: реферальный %, white-label.
3. **Insta + UGC от организаторов** — кейсы реальных свадеб с скринами платформы. KZ — особенно сильный канал.
4. **Telegram-каналы** (TJ свадебные/ивент-каналы) — органика через гостевые посты, реклама в крупных каналах.
5. **Контент для SEO** — рано, но «как организовать свадьбу на 300 человек» / «чек-лист корпоратива» — холодная воронка с долгим эффектом.
6. **Платный трафик (Google/Meta Ads)** — НЕ сейчас. Только когда конверсия лендинга подтверждена органикой.

## 8. Чек-лист GTM-брифа фичи

Перед тем как считать бриф готовым, проверь:

- [ ] Назван конкретный ICP (сегмент + рынок), не «все организаторы»
- [ ] Сформулирована минимум одна JTBD по канонической формуле
- [ ] Есть Value Proposition по формуле из раздела 3
- [ ] Указан минимум один конкретный конкурент и в чём наше отличие
- [ ] Каналы продвижения — 1-3, не «везде»
- [ ] Привязка к сезонному окну (если применимо)
- [ ] Заданы метрики успеха через 30/60/90 дней
- [ ] TJ и KZ разделены если фича рассчитана на оба рынка
- [ ] Если у фичи нет JTBD ни в одном сегменте — флажок «продакт-маркетинг сомнителен»

## 9. Анти-паттерны (отклоняй сразу)

1. **«Маркетинг для всех»** — без сегмента. Отправляй на доработку с ICP.
2. **«Запустим рекламу»** — без понимания канала/ICP/сезона. Отправляй на гипотезу.
3. **«Сделаем как Eventbrite»** — копирование без сегмента. Eventbrite это билетинг для B2C ивентов с открытой регистрацией — это не Meet.
4. **«Скидку дадим — купят»** — цена обычно не главная боль в event-tech. Сначала проверь JTBD.
5. **«Контент-маркетинг» без воронки** — куда ведёт контент, какой CTA, какой next step. Если нет ответа — нет смысла.
6. **«Усилим бренд»** — без метрики. Бренд это следствие повторяющихся правильных касаний с ICP, а не цель.
