# UX/UI Design Best Practices

## Design System (единый источник правды)
Все визуальные решения берутся ТОЛЬКО из design-system.md.
ЗАПРЕЩЕНО использовать значения цветов, шрифтов, отступов
вне переменных дизайн-системы.

## UX принципы

### Иерархия и навигация
- Пользователь понимает где он находится за 3 секунды
- Основное действие на экране всегда одно (Primary CTA)
- Не более 7 элементов в навигации (закон Миллера)
- Хлебные крошки для вложенности глубже 2 уровней

### Формы и ввод данных
- Метки над полями, не внутри (placeholder исчезает при вводе)
- Валидация inline — сразу после потери фокуса, не после submit
- Группировать связанные поля визуально
- Кнопка submit активна всегда, ошибки показывать явно

### Обратная связь и состояния
- Каждый интерактивный элемент имеет 4 состояния:
  default → hover → active → disabled
- Loading state для всех async операций (скелетон или спиннер)
- Empty state с подсказкой что делать дальше
- Error state с чётким объяснением и способом исправить
- Success state подтверждает что действие выполнено

### Доступность (a11y)
- Контраст текста минимум 4.5:1 (WCAG AA)
- Все интерактивные элементы доступны с клавиатуры
- aria-label для иконок без текста
- Focus visible — видимый outline при навигации с клавиатуры
- Размер touch target минимум 44x44px

## UI принципы

### Типографика
- Максимум 2 шрифта в проекте (heading + body)
- Размеры по шкале: 12, 14, 16, 18, 20, 24, 32, 40, 48
- Line-height для body: 1.5, для heading: 1.2
- Максимальная ширина строки: 65-75 символов (ch)

### Отступы и сетка
- Базовая единица: 4px (всё кратно 4)
- Отступы по шкале: 4, 8, 12, 16, 24, 32, 48, 64
- Консистентные внутренние отступы в компонентах одного типа

### Цвет
- Primary — основное действие (1 цвет)
- Secondary — вспомогательное действие
- Semantic цвета: success (#green), warning (#yellow), error (#red), info (#blue)
- Нейтральные: минимум 5 оттенков серого
- Dark mode: определить заранее если планируется

### Компоненты
- Атомарный дизайн: atoms → molecules → organisms → templates
- Каждый компонент работает изолированно
- Состояния компонента задокументированы
- Responsive: mobile-first подход

## Применение в Angular (текущий проект)
- Токены дизайн-системы — CSS-переменные в ui-ng/src/styles/tokens.css
- SCSS-переменные в ui-ng/src/styles.scss (уже существует)
- Компоненты в ui-ng/src/app/shared/components/ — переиспользуемые
- Angular Material или кастомные компоненты — без смешивания стилей
- Все размеры через CSS-переменные или SCSS-переменные, не хардкод

## Чеклист перед сдачей UI
- [ ] Все цвета из переменных дизайн-системы
- [ ] Все отступы кратны 4px
- [ ] Есть все 4 состояния интерактивных элементов
- [ ] Контраст проверен (минимум 4.5:1)
- [ ] Работает на мобильном (320px минимум)
- [ ] Нет хардкода значений вне дизайн-системы
