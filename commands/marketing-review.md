Используй агента event-marketer для маркетингового анализа: $ARGUMENTS

Режимы вызова:
- `/project:marketing-review <фича>` — GTM-бриф для конкретной фичи (Режим 1). Агент сохранит результат в `docs/marketing/<feature>-gtm.md`.
- `/project:marketing-review <вопрос>` — свободный анализ (ICP / конкурент / сегмент / сезонное окно). Агент ответит в чате без сохранения файла.

Передай агенту полный текст $ARGUMENTS.

Перед вызовом агента собери **PRE-FLIGHT CONTEXT** и инжектируй в prompt:
- Если задача про конкретную фичу — найди связанный ADR в `docs/adr/`, UI Spec в `docs/ui-specs/`, упоминания в `docs/API_CONTRACT.md`
- Найди связи с `Feature Restriction Matrix` (какие тарифы фича затрагивает)
- Проверь нет ли уже GTM-брифа в `docs/marketing/` по этой теме (не дублируй)

Агент должен:
1. Определить сегмент (свадьбы / корпоративы / конференции / ...) и рынок (TJ / KZ / оба)
2. Сформулировать JTBD по канонической формуле
3. Дать Value Proposition по формуле из event-marketing.md
4. Указать конкурентное отличие
5. Предложить 1-3 канала продвижения с обоснованием
6. Привязать к сезонному окну
7. Задать метрики успеха через 30/60/90 дней
8. Пройти чек-лист GTM-брифа (раздел 8 скилла)

Если у фичи нет внятной JTBD ни в одном сегменте — агент обязан явно отметить это как **флаг для пересмотра продакт-решения**, не маскировать.

После получения результата — выведи ответ пользователю как есть. Это обсуждение, пользователь может уточнять, просить переписать под другой сегмент, оспорить позиционирование.
